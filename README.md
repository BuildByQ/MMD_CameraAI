MMDカメラ生成AI ― 設計フェイズまとめ（完成版）

０. 今回作成したいもの

MMDで動画制作をしていて、自分でカメラモーションを作る機会がありました。しかし、要点をつかむのが難しく、作成にかなりの時間がかかって負担が大きかったため、その負担軽減を目的にMMDのカメラモーションを自動生成するプログラムを作成しようと考えています。

人間が作成した「音源（wav）・被写体のモーション（vmdをcsvに変換）・カメラモーション（vmdをcsvに変換）」を用いて学習モデルを作成します。

学習したモデルは「音源・被写体のモーション」を入力として受け取り、カメラモーションファイルを出力します。

出力されたカメラモーションは、MMDでの動画制作時に使用します。

今回作成するものは試作版であり、今後より精度の高いモデルを目指して拡張していく予定です。

1. 入力特徴量（決定済み）

距離: log変換（復元時は exp⁻¹、負値は0クリップ）
Yaw: sin/cos変換＋回転回数 one-hot（-2〜+2）
Pitch／Roll: 角度のみ、回転回数は持たない
FOV: 範囲固定（1〜125）、正規化方式は選択式（min-max／z-score／生値）

2. 演出ラベル（決定済み）
方式: ソフトラベル＋マルチラベル
減衰: 線形減衰（閾値・マージンは設定ファイルで外部指定）
ラベルセット: 過去に合意したラベル群（例示のロー／チルト／パン／ズームはサンプル）

3. 境界（決定済み）
判定方式: ラベルベクトル差分（ユークリッド距離）による後処理判定
分類: 強境界／弱境界／なし
閾値: 設定ファイルで外部指定（開発段階で統計に基づき調整）
オプション: スムージング、デバウンス、ウィンドウ比較（設定で制御）

4. 出力仕様（決定済み）

シーン: キーフレーム（始点・中間点・終点）
カメラ: 位置・角度・距離・FOV
ラベル: 各フレームのソフト＋マルチラベル

5. 学習方法（決定済み）

第1モデル（ラベル出力）

意図：座標や補間曲線を使って「前後の動き」を捉え、その時点で最適なカメラワーク=演出ラベルを分類する。
演出ラベルとは：ロー／チルト／パン／ズームなどのカメラワークの分類。第二モデルは第一モデルが導いた演出ラベルを用いてカメラモーションを作成する。

入力: 前後5フレームの特徴量、音声データ（MFCC）、ボーンモーション（補完関数を用いてフレームを全部埋めたもの）、演出ラベル（フレームを全部埋めたもの）

出力: 全フレームに全演出ラベルの値が入った演出ラベルベクトル

方式: 教師あり学習（既存カメラモーションを教師データ化）

境界: ラベル差分から後処理で判定

第2モデル（座標生成）

入力: 演出ラベルベクトル＋姿勢・曲調などの特徴量

出力: カメラ座標（位置・角度・距離・FOV）

方式: 教師あり学習を基本とするが、正解座標が唯一でないため補助損失を導入

ラベル適合性

滑らかさ

構図ヒューリスティクス

補助損失の具体的な計算式や重み付けは開発段階で調整予定。初期は軽めに実装し、出力結果を見ながら調整を行う方針。

将来的拡張: GANや強化学習的枠組みを検討余地として残す。これは、人手での補助損失計算式設計の難しさを補うための検討である。

6. 評価方法（決定済み）

第1モデル

主要指標: ラベル一致率（閾値を超えたラベルの正解率）

補助指標: ラベルベクトル誤差（MSE/MAE）、境界検出精度（F1）

定性評価: MMD再生で演出切り替えの自然さを確認

評価統合: ラベルはすべて等価で重み付けなし。多次元の0～1のベクトルとして扱い、試作版として目的を果たす。

定性評価の役割: 試作版から次版へのフィードバックや学習・出力時の閾値調整に活用。

第2モデル

主要指標: 意図整合性・滑らかさ・構図・同期度などのプロキシ指標群

補助指標: 違和感ペナルティ（急な回転反転、FOV飛びなど）

定性評価: MMD再生で自然さ・違和感を確認

将来的拡張: GAN判別器による「人間的かどうか」の自動評価を検討

評価方法補足

第1モデルのラベルはカメラワーク（ローアングル、パンなど）で、すべて等価の多次元ベクトルとして扱う。

定性的評価は主に次版へのフィードバックや閾値調整に使われる。

複数指標の重み付けは設計段階では特に設定せず、試作版の運用で調整予定。

第2モデルの主要指標は等価ではなく、0～1で値を返す関数を各指標ごとに作成し、統合しやすくする方針。

定性評価は第1モデルと同様に次版へのフィードバックや閾値調整に活用。

7. 学習データ準備（決定済み）

方針: 既存カメラモーションから自動抽出

方法: XYZ座標・回転・距離・FOVを判定関数でラベル化

出力: ソフトラベル＋マルチラベル教師データ

調整: 閾値・マージンは設定ファイルで外部指定

人手作業: 原則不要（検証用サンプルのみ人手確認）

8. 将来的拡張（決定済み）

方針: 試作版を終えて出力結果次第で取り入れます。GANのほうが計算時間や調整が難しいと聞いているので、それ次第です。ただ取り換えが容易なように開発したいと思っています。

設計フェイズの総括

決定済み: 入力特徴量、ラベル方式、境界判定、出力仕様、学習方法（第1・第2モデル枠組み）、評価方法（主要指標含む）、学習データ準備

保留／開発段階で調整: 閾値の具体値、補助損失の詳細、評価指標の重み付け

将来的拡張: GAN判別器による評価導入
